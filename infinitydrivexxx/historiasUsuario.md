## Requerimientos funcionales y no funcionales

### Requerimientos funcionales

+ El sistema debe generar obstáculos de diferentes tipos en cada partida.  
+ Los obstáculos deben aparecer en posiciones aleatorias para garantizar variabilidad.  
+ Algunos obstáculos deben moverse dinámicamente en la pista.  
+ La base de datos debe almacenar información sobre jugadores, partidas y mejoras.  
+ Se deben incluir mejoras temporales y permanentes para los autos.  
+ Las mejoras deben aplicarse correctamente sin errores en la jugabilidad.  
+ Los precios de las mejoras deben estar bien balanceados dentro del juego.  
+ La interfaz del usuario debe ser explicada dentro del juego.  
+ Las animaciones de movimiento deben ser fluidas y sin errores visuales.  
+ Se deben incluir efectos visuales en colisiones y aceleraciones.  
+ Todo el entorno debe ser representado en dos dimensiones.  
+ Los elementos deben respetar la física de un plano bidimensional.  
+ Los controles deben ser intuitivos y explicados al inicio del juego.  
+ Debe tener un nivel de tutorial.  
+ Se deben realizar pruebas de calidad antes de cada actualización.  
+ El juego debe registrar datos de cada partida, como tiempo jugado y mejoras obtenidas.  
+ Las estadísticas deben poder visualizarse en un menú.  
+ Los niveles deben volverse más difíciles conforme avanza la partida.  
+ Deben agregarse más obstáculos.  

### Requerimientos no funcionales

+ Los nombres de las tablas deben ser claros y lógicos.  
+ Los autos deben tener un estilo visual animado y caricaturesco.  
+ Se debe permitir la personalización de los controles.  
+ Los autos deben tener diferentes skins desbloqueables.  
+ Cada acción importante debe tener un efecto de sonido.  
+ El tiempo de carga no debe exceder los 5 segundos.  
+ El juego debe funcionar en Windows, macOS y Linux.  
+ Debe ser compatible con Chrome, Firefox y Edge.  
+ Hacer el código del videojuego organizado para encontrar las cosas más fácilmente y tener un orden.  
+ La interfaz debe ser fácil de usar y accesible.  
+ Los botones deben estar bien organizados y ser visibles.  
+ La arquitectura del juego debe permitir la adición de nuevos niveles y autos sin afectar el código base.  
+ El juego debe incluir música de fondo con opción de desactivarla.  
+ Debe haber un sistema de respaldo de datos automático cada 24 horas.  
+ El juego debe ajustarse a diferentes tamaños de pantalla sin distorsionarse.  
+ La interfaz de estadísticas debe ser clara y comprensible para todos los jugadores.  
+ Recoger monedas debe generar un sonido y una animación distintiva para mejorar la experiencia del jugador.  
+ Las habilidades deben funcionar de la misma manera en todos los dispositivos compatibles.  
+ La pantalla de inicio debe adaptarse a diferentes resoluciones y dispositivos sin perder calidad.  
+ Las superficies del juego deben afectar la conducción sin hacer el juego injustamente difícil.
